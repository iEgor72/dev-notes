С чего начать

Layout — это способ расположить элементы на странице: в строку, в колонку, в сетку, по центру, по краям или поверх других блоков.

Для новичка самое важное правило такое: основу макета чаще всего строят через Flexbox и Grid, а position используют точечно, когда нужно закрепить, наложить или сместить элемент.

Хороший layout — это не просто красивая раскладка блоков, а предсказуемое поведение интерфейса на разных размерах экрана.

Когда выбирать Flexbox, Grid и Position

Какой инструмент layout использовать
Инструмент Когда подходит лучше всего
Flexbox Когда элементы нужно расположить в одну строку или в один столбец и удобно ими управлять
Grid Когда нужен макет с колонками и рядами: карточки, секции, галереи, общая структура страницы
Position Когда элемент нужно наложить поверх другого, закрепить, сместить или сделать прилипающим
  • Flexbox — одномерный layout: строка или колонка.
  • Grid — двумерный layout: колонки и ряды одновременно.
  • Position — вспомогательный инструмент, а не основа всей раскладки.

Частая практика в реальных проектах: Grid для общей структуры и Flexbox внутри отдельных блоков.

Flexbox

CSS
.container { display: flex; justify-content: center; align-items: center; gap: 20px; }

Flexbox удобен, когда нужно расположить элементы в одну строку или в один столбец и контролировать расстояния, направление и выравнивание.

У Flexbox есть две оси:

  • главная ось — определяется свойством flex-direction
  • поперечная ось — идёт поперёк главной

Если flex-direction: row, элементы идут в строку:

  • justify-content работает по горизонтали
  • align-items работает по вертикали

Если flex-direction: column, логика меняется:

  • justify-content работает по вертикали
  • align-items работает по горизонтали
Частые свойства Flexbox
Свойство Что делает
display: flex Включает flex-контекст
flex-direction Задаёт направление: строка или столбец
justify-content Распределяет элементы по главной оси
align-items Выравнивает элементы по поперечной оси
gap Добавляет расстояние между элементами
flex-wrap Разрешает перенос элементов на новую строку
CSS
.cards { display: flex; flex-wrap: wrap; gap: 16px; }

flex-wrap полезен, когда элементов много и им нужно переноситься, а не пытаться уместиться в одну линию.

Flexbox особенно хорош для меню, кнопок, шапок, панелей, выравнивания элементов внутри карточек и небольших интерфейсных блоков.

Grid

CSS
.container { display: grid; grid-template-columns: repeat(3, 1fr); gap: 20px; }

Grid удобен, когда нужно строить сетки: несколько колонок, ряды, карточки, галереи и более сложные макеты.

  • display: grid — включает сетку
  • grid-template-columns — задаёт колонки
  • grid-template-rows — задаёт ряды
  • gap — расстояние между ячейками
  • grid-column — управляет шириной по колонкам
  • grid-row — управляет положением по рядам
Частые свойства Grid
Свойство Что делает
display: grid Включает CSS Grid
grid-template-columns Задаёт количество и размер колонок
grid-template-rows Задаёт ряды
gap Добавляет расстояние между ячейками
grid-column Позволяет элементу занять несколько колонок
grid-row Позволяет элементу занять несколько рядов
CSS
.cards { display: grid; grid-template-columns: repeat(auto-fit, minmax(240px, 1fr)); gap: 20px; }

Такой вариант особенно удобен для адаптивных карточек. minmax(240px, 1fr) задаёт минимальную ширину карточки, а auto-fit помогает сетке автоматически перестраиваться по ширине экрана.

Grid чаще всего удобнее для макета секции или страницы, а Flexbox — для расположения элементов внутри отдельного блока.

Position

Популярные значения свойства position
Значение Описание
static Обычное положение элемента по умолчанию
relative Позволяет сдвигать элемент относительно обычного места
absolute Позиционирует элемент относительно ближайшего positioned-родителя
fixed Фиксирует элемент относительно окна браузера
sticky Элемент «прилипает» при прокрутке

Важно: absolute и fixed часто вырывают элемент из нормального потока документа. Остальные блоки ведут себя так, как будто этого элемента нет.

CSS
.parent { position: relative; } .badge { position: absolute; top: 12px; right: 12px; }

Если используешь position: absolute, родителю часто задают position: relative, чтобы дочерний элемент позиционировался внутри него, а не относительно всей страницы.

Не строй весь layout через position. Для основной структуры почти всегда лучше использовать Flexbox и Grid.

Контейнеры и ширина

CSS
.container { width: 100%; max-width: 1200px; margin: 0 auto; padding: 0 20px; }

Контейнер помогает ограничить ширину контента, чтобы текст и блоки не растягивались слишком широко на больших экранах.

  • width: 100% — блок занимает доступную ширину
  • max-width — ограничивает максимальную ширину
  • margin: 0 auto — центрирует контейнер
  • padding — даёт внутренние отступы по краям

Без ограничения ширины длинный текст становится труднее читать, особенно на широких экранах.

Box sizing

CSS
*, *::before, *::after { box-sizing: border-box; }

Это правило помогает предсказуемо считать размеры элементов. Внутренние отступы и границы включаются в итоговую ширину и высоту блока.

Для новичков это особенно полезно, потому что layout начинает вести себя намного понятнее.

У тебя это правило уже есть в глобальных стилях — и это хорошее решение для всего проекта.

Выравнивание

В layout чаще всего нужно управлять двумя вещами: положением элементов и расстояниями между ними.

Частые способы выравнивания
Инструмент Для чего нужен
justify-content Распределение элементов по главной оси
align-items Выравнивание элементов по поперечной оси
gap Расстояние между элементами
margin: 0 auto Центрирование блочного элемента по горизонтали
text-align: center Выравнивание текста и inline-элементов
place-items: center Быстрое центрирование содержимого в Grid
CSS
.hero-box { display: flex; flex-direction: column; align-items: center; gap: 16px; text-align: center; }

Частые задачи по центрированию:

  • текст по центру — text-align: center
  • блок по центру страницы — margin: 0 auto
  • элемент по центру внутри контейнера — Flexbox или Grid
CSS
.center-box { display: grid; place-items: center; }

Адаптивность

CSS
@media (max-width: 768px) { .grid { grid-template-columns: 1fr; } }

Media queries используются для адаптации сайта под разные размеры экрана: мобильные устройства, планшеты и большие мониторы.

  • уменьшать количество колонок на маленьких экранах;
  • перестраивать навигацию и карточки;
  • избегать горизонтальной прокрутки;
  • не злоупотреблять жёсткими фиксированными ширинами;
  • проверять, как блоки ведут себя при сужении экрана.
CSS
img { max-width: 100%; height: auto; }

Это полезное правило для изображений: они не будут вылезать за пределы родителя на маленьких экранах.

Хороший адаптивный интерфейс не просто «сжимается», а перестраивается так, чтобы им было удобно пользоваться на любом устройстве.

Частые паттерны layout

Что обычно выбирать для типичных задач
Задача Что обычно подходит
Меню в одну строку Flexbox
Карточки в несколько колонок Grid
Адаптивные карточки Grid + auto-fit/minmax()
Центрирование блока Flexbox, Grid или margin: 0 auto
Бейдж поверх карточки Position absolute + relative
Прилипающий заголовок position: sticky
Адаптация под мобильные Media queries + гибкие размеры

Частые ошибки новичков

Типичные ошибки при работе с layout
Ошибка Почему плохо Как лучше
Пытаться строить весь layout только через position Макет становится хрупким и сложным в поддержке Сначала использовать Flexbox и Grid, а position — точечно
Не ограничивать ширину контента Текст трудно читать на широких экранах Использовать контейнер с max-width
Не проверять мобильную версию Интерфейс может ломаться на маленьких экранах Добавлять media queries и тестировать responsive
Использовать случайные margin везде Структура становится непредсказуемой Строить систему отступов осознанно
Не использовать gap в flex/grid Приходится вручную задавать много лишних отступов По возможности использовать gap
Задавать слишком много фиксированных ширин Макет хуже адаптируется под разные экраны Чаще использовать гибкие размеры и max-width

Плохо

CSS
.card { position: absolute; left: 100px; top: 200px; }

Лучше

CSS
.cards { display: grid; grid-template-columns: repeat(auto-fit, minmax(240px, 1fr)); gap: 20px; }

Что важно запомнить

  • Flexbox удобен для строки или столбца
  • Grid удобен для сеток и более сложной структуры
  • position нужен точечно, а не для всего layout
  • absolute и fixed часто вырывают элемент из потока документа
  • Контейнер с max-width делает страницу удобнее для чтения
  • gap помогает делать чистые и понятные расстояния
  • box-sizing: border-box делает размеры блоков предсказуемее
  • Адаптивность — обязательная часть современного интерфейса
