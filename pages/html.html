Что такое HTML

HTML — это язык разметки, который задаёт структуру страницы: заголовки, текст, изображения, ссылки, таблицы, формы и смысловые блоки.

HTML отвечает за содержимое и структуру, CSS — за оформление, JavaScript — за поведение страницы.

HTML нужен не для того, чтобы сделать страницу красивой, а для того, чтобы правильно описать её структуру и смысл.

Как работает веб-страница

Современная веб-страница обычно состоит из трёх основных технологий:

  • HTML — структура и содержимое
  • CSS — внешний вид и оформление
  • JavaScript — поведение и интерактивность

HTML описывает, что находится на странице. CSS определяет, как это выглядит. JavaScript добавляет логику и действия.

Как устроены HTML-элементы

Большинство HTML-элементов состоит из трёх частей:

  • открывающий тег
  • содержимое
  • закрывающий тег
<p>Это текст внутри абзаца</p>

У элементов могут быть атрибуты — это дополнительные настройки, которые уточняют поведение элемента. 

<a href="https://example.com">Ссылка</a>

В примере выше href — это атрибут, а его значение — адрес ссылки.

Не все элементы имеют закрывающий тег. Например, <img>, <br> и <hr> — пустые элементы.

Базовая структура HTML 

<!DOCTYPE html>
    <html lang="ru">
    <head>
      <meta charset="UTF-8" />
      <meta name="viewport" content="width=device-width, initial-scale=1.0" />
      <title>Название страницы</title>
    </head>
    <body>
      Содержимое страницы
    </body>
    </html>
Основные части HTML-документа
Часть Что делает
<!DOCTYPE html> Сообщает браузеру, что это HTML5-документ
<html lang="ru"> Корневой элемент страницы, атрибут lang задаёт язык документа
<head> Содержит служебную информацию: кодировку, заголовок, метатеги, подключение стилей
<body> Содержит всё, что видит пользователь на странице

head нужен для служебной информации, а основной видимый контент страницы размещается внутри body.

Заголовки и текст

Заголовки создают структуру страницы. Они должны идти по порядку, как главы и подглавы в книге.

Базовые теги для заголовков и текста
Тег Для чего нужен
<h1> Главный заголовок страницы
<h2> Заголовок раздела
<h3> Заголовок подраздела
<p> Обычный абзац текста

Не выбирай заголовок по размеру текста. Размер меняют через CSS, а уровень заголовка нужен для правильной структуры документа. 

<h1>Developer Notes</h1>
    <h2>HTML</h2>
    <h3>Ссылки</h3>
    <p>Это обычный абзац текста.</p>

Блочные и строчные элементы

HTML-элементы часто делят на два основных типа: блочные и строчные.

Блочные элементы

Обычно занимают всю доступную ширину и начинаются с новой строки. 

<h1>
    <p>
    <div>
    <section>

Строчные элементы

Располагаются внутри строки и занимают только нужную ширину.

<span>
    <a>
    <strong>
    <em>

Это полезно знать, когда ты начинаешь писать CSS и управлять расположением элементов.

Это упрощённая модель для старта. В CSS поведение элемента также зависит от свойства display.

Основные HTML-теги

Часто используемые HTML-теги
Тег Что делает Когда использовать Пример
<h1> – <h6> Заголовки Для названия страницы и разделов <h1>Мой сайт</h1>
<p> Абзац Для обычного текста <p>Это абзац текста.</p>
<a> Ссылка Для переходов на другие страницы и сайты <a href="https://example.com">Сайт</a>
<img> Изображение Для картинок и иллюстраций <img src="cat.jpg" alt="Кот" />
<div> Блочный контейнер без собственного смысла Для группировки элементов <div>Блок</div>
<span> Строчный контейнер без собственного смысла Для выделения части текста <span>Выделение</span>
<strong> Важный текст Для смыслового выделения <strong>Важно</strong>
<em> Смысловой акцент в тексте Для логического выделения <em>Обрати внимание</em>
<br> Перенос строки Когда нужен разрыв строки внутри текста Первая строка<br>Вторая строка
<hr> Тематическое разделение контента Когда нужно логически отделить одну часть от другой <hr>

Пример простого контента 

<h1>Добро пожаловать</h1>
    <p>Это моя первая HTML-страница.</p>
    <a href="https://developer.mozilla.org/" target="_blank" rel="noopener noreferrer">
      Изучать HTML на MDN
    </a>

Списки

Списки бывают двух основных типов: маркированные и нумерованные.

Элементы списка всегда записываются в теге <li>, который должен находиться внутри <ul> или <ol>.

Маркированный список

<ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>

Нумерованный список

<ol>
      <li>Изучить HTML</li>
      <li>Изучить CSS</li>
      <li>Изучить JavaScript</li>
    </ol>

Ссылки

Тег <a> используется для переходов между страницами, сайтами, разделами страницы, а также для email- и телефонных ссылок. 

<a href="https://google.com">Открыть Google</a>
    <a href="./about.html">Перейти на страницу About</a>
    <a href="#contacts">Перейти к контактам</a>
    <a href="mailto:hello@example.com">Написать письмо</a>
    <a href="tel:+41791234567">Позвонить</a>
Частые атрибуты ссылок
Атрибут Назначение
href Адрес ссылки
target="_blank" Открывает ссылку в новой вкладке
rel="noopener noreferrer" Повышает безопасность внешних ссылок, открывающихся в новой вкладке
title Подсказка при наведении; не должна быть единственным источником важной информации

Для внешних ссылок, которые открываются в новой вкладке, желательно добавлять rel="noopener noreferrer".

Лучше писать понятный текст ссылки, например «Документация MDN», а не просто «Нажми сюда».

Изображения

Тег <img> вставляет изображение на страницу. 

<img src="images/photo.jpg" alt="Описание изображения" width="300" height="200" />
Частые атрибуты изображения
Атрибут Назначение
src Путь к изображению
alt Альтернативный текст: важен для доступности, экранных читалок и случаев, когда картинка не загрузилась
width / height Ширина и высота изображения; помогают браузеру заранее выделить место под картинку

Атрибут alt особенно важен. Он помогает:

  • если изображение не загрузилось;
  • пользователям экранных читалок;
  • лучше понимать содержимое изображения.

Если изображение декоративное и не несёт смысла, обычно используют пустой alt="".

Контейнеры div и span

Теги <div> и <span> не имеют собственного смыслового значения. Они используются как контейнеры для группировки элементов и стилизации.

div — обычно блочный контейнер.

<div class="card">
      <h2>Заголовок</h2>
      <p>Описание карточки</p>
    </div>

span — обычно строчный контейнер.

<p>Это <span>важная часть</span> текста.</p>

Если элемент имеет смысловое значение, лучше использовать семантический тег, а не просто div.

Формы и кнопки

Формы нужны для отправки данных: логин, регистрация, поиск, обратная связь. 

<form action="/submit" method="post">
      <label for="name">Имя</label>
      <input type="text" id="name" name="name" autocomplete="name" placeholder="Введите имя" required />

      <label for="email">Email</label>
      <input type="email" id="email" name="email" autocomplete="email" placeholder="Введите email" required />

      <label for="message">Сообщение</label>
      <textarea id="message" name="message" rows="4" placeholder="Введите сообщение"></textarea>

      <button type="submit">Отправить</button>
    </form>
Основные элементы формы
Тег Для чего нужен
<form> Обёртка для полей формы
<input> Поле ввода
<label> Подпись к полю
<button> Кнопка отправки или действия
<textarea> Поле для большого текста
<select> Выпадающий список

label — это название поля.
placeholder — это подсказка внутри поля.

placeholder не должен заменять label. Поле должно быть понятно пользователю даже без текста-подсказки.

Атрибут name нужен для отправки данных формы на сервер.

Для полей формы часто полезны атрибуты autocomplete, required и понятные тексты ошибок.

Популярные типы input

Популярные типы поля input
Тип Для чего используется Пример
text Обычное текстовое поле для ввода данных <input type="text" placeholder="Введите имя">
email Поле для email-адреса со встроенной проверкой формата <input type="email" placeholder="example@mail.com">
password Поле для пароля — символы скрываются <input type="password" placeholder="Введите пароль">
number Поле для ввода чисел <input type="number" min="1" max="100">
tel Поле для номера телефона <input type="tel" placeholder="+41 79 123 45 67">
url Поле для ввода ссылки <input type="url" placeholder="https://example.com">
date Выбор даты через встроенный календарь <input type="date">
range Ползунок для выбора значения из диапазона <input type="range" min="0" max="100">
color Выбор цвета через встроенный color picker <input type="color">
checkbox Флажок для выбора одного или нескольких вариантов <label><input type="checkbox"> Я согласен</label>

Семантические теги

Семантические теги помогают браузеру, поисковикам и разработчикам понимать структуру страницы. Они предпочтительнее, чем множество безымянных <div>.

Популярные семантические теги
Тег Назначение
<header> Шапка сайта или секции
<nav> Навигация
<main> Основное содержимое страницы
<section> Смысловой раздел страницы
<article> Независимый блок контента: статья, пост, карточка
<aside> Дополнительная или боковая информация
<footer> Подвал сайта или секции 
<header>
      <nav>Меню сайта</nav>
    </header>

    <main>
      <section>
        <h2>Новости</h2>
        <article>
          <h3>Вышла новая статья</h3>
          <p>Краткое описание новости...</p>
        </article>
      </section>
    </main>

    <footer>Подвал сайта</footer>

Хорошее правило: если элемент можно назвать по смыслу, лучше выбрать семантический тег, а не div.

Полезные атрибуты

Атрибуты — это дополнительные настройки тега. Они пишутся в открывающем теге и уточняют поведение элемента.

Часто используемые HTML-атрибуты
Атрибут Что делает Пример
class Задаёт CSS-класс <div class="card"></div>
id Уникальный идентификатор элемента <section id="about"></section>
title Подсказка при наведении <a title="Подробнее">...</a>
alt Альтернативный текст для изображений <img alt="Логотип" />
placeholder Подсказка внутри input <input placeholder="Введите имя" />
name Имя поля при отправке формы <input name="email" />
disabled Делает элемент неактивным <button disabled>Недоступно</button>
required Делает поле обязательным <input required />
autocomplete Помогает браузеру подставлять данные пользователя <input autocomplete="email" />
aria-label Добавляет доступное имя, когда обычного текста недостаточно <nav aria-label="Основная навигация"></nav>

Частые ошибки новичков

Типичные ошибки при работе с HTML
Ошибка Почему плохо Как правильно
Использовать только <div> везде Страница теряет смысловую структуру Использовать <header>, <main>, <section>, <footer>
Не указывать alt у картинки Плохо для доступности и понимания изображения Добавлять понятное описание изображения
Пропускать <label> у формы Поля менее понятны пользователю Связывать label с input через for и id
Ставить несколько <h1> без необходимости Путает структуру документа Делать логичную иерархию заголовков
Открывать внешнюю ссылку в новой вкладке без rel="noopener noreferrer" Это менее безопасно Добавлять rel="noopener noreferrer"
Использовать <br> вместо абзацев Текст становится менее структурированным Для отдельных абзацев использовать <p>
Делать кнопку из <div> Элемент теряет правильное поведение и смысл Использовать <button> для действий
Не указывать lang="ru" у документа Это ухудшает доступность и понимание языка страницы Указывать язык в теге <html>

Плохо

<div>Отправить</div>

Лучше

<button>Отправить</button>

Плохо

Текст<br>
    Текст<br>
    Текст

Лучше

<p>Текст</p>
    <p>Текст</p>

Что важно запомнить

  • HTML отвечает за структуру страницы
  • CSS отвечает за внешний вид
  • JavaScript отвечает за поведение
  • Используй семантические теги там, где есть смысл
  • Добавляй alt для изображений
  • Связывай label и input в формах
  • Не выбирай заголовки по размеру — выбирай их по смыслу
  • div и span — это контейнеры, а не смысловые элементы

Мини-пример готовой HTML-страницы 

<body>
      <a href="#main-content" class="skip-link">Перейти к содержимому</a>

      <header>
        <h1>Мой сайт</h1>
        <nav aria-label="Основная навигация">
          <a href="#about">Обо мне</a>
          <a href="#contacts">Контакты</a>
        </nav>
      </header>

      <main id="main-content">
        <section id="about">
          <h2>Обо мне</h2>
          <p>Я начинающий frontend-разработчик.</p>
        </section>

        <section id="contacts">
          <h2>Контакты</h2>
          <a href="mailto:test@mail.com">test@mail.com</a>
        </section>
      </main>

      <footer>
        <p>2026 © Мой сайт</p>
      </footer>
    </body>
