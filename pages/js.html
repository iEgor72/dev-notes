Что такое JavaScript

JavaScript — это язык программирования, который добавляет страницам поведение, логику и интерактивность.

С его помощью можно реагировать на действия пользователя, менять содержимое страницы, работать с данными, отправлять запросы на сервер и строить полноценные приложения.

HTML отвечает за структуру, CSS — за внешний вид, JavaScript — за поведение.

Подключение JavaScript 

<script src="script.js"></script>

Внешний файл JavaScript обычно подключают перед закрывающим тегом </body>, чтобы HTML успел загрузиться до выполнения скрипта. 

<script src="script.js" defer></script>

Атрибут defer часто удобнее: скрипт загружается параллельно, а выполняется после разбора HTML.

Переменные

let name = "Alex";
const age = 25;
var oldWay = "устаревший вариант";
  • let — переменная, значение можно менять
  • const — константа, её нельзя переназначить
  • var — старый способ объявления переменных

В современном коде обычно используют const по умолчанию и let, если значение должно меняться.

Операторы

const a = 10;
const b = 5;

console.log(a + b);
console.log(a > b);
console.log(a === 10);
console.log(a !== b);

Часто используются такие группы операторов:

  • арифметические: +, -, *, /
  • сравнения: >, <, >=, ===, !==
  • логические: &&, ||, !

Обычно лучше использовать строгое сравнение ===, а не ==.

Условия

if (age >= 18) {
  console.log("adult");
} else {
  console.log("minor");
}
const message = age >= 18 ? "adult" : "minor";

Условия помогают выполнять разный код в зависимости от значения.

Циклы

for (let i = 0; i < 5; i++) {
  console.log(i);
}
for (const item of ["HTML", "CSS", "JS"]) {
  console.log(item);
}

Циклы нужны, когда один и тот же код надо повторить несколько раз.

Функции

function greet(name) {
  return "Hello " + name;
}

const sum = (a, b) => a + b;

Функции позволяют переиспользовать код, разбивать программу на части и делать логику понятнее.

Популярные виды функций
Вид Пример
Обычная функция function sayHi() {}
Функциональное выражение const sayHi = function() {}
Стрелочная функция const sayHi = () => {}

Массивы

const fruits = ["apple", "banana", "orange"];

console.log(fruits[0]);
fruits.push("grape");

Массив — это список значений, к которым можно обращаться по индексу.

Полезные методы массива
Метод Что делает
push() Добавляет элемент в конец
pop() Удаляет последний элемент
map() Создаёт новый массив, преобразуя каждый элемент
filter() Оставляет только подходящие элементы
find() Ищет первый подходящий элемент

Объекты

const user = {
  name: "Alex",
  age: 25,
  isAdmin: false
};

console.log(user.name);

Объект хранит данные в формате ключ: значение. 

const { name, age } = user;

Такая запись называется деструктуризацией. Она часто используется в современном JavaScript.

Работа с DOM

const title = document.querySelector("h1");

title.textContent = "Hello JS";
  • querySelector() — находит первый подходящий элемент
  • querySelectorAll() — находит несколько элементов
  • textContent — меняет текст
  • classList.add() — добавляет класс
  • classList.remove() — удаляет класс
const box = document.querySelector(".box");
box.classList.add("active");

События

const button = document.querySelector("button");

button.addEventListener("click", () => {
  console.log("clicked");
});

Частые события:

  • click — клик
  • input — ввод в поле
  • submit — отправка формы
  • change — изменение значения
  • keydown — нажатие клавиши

Async / Await и Fetch 

async function loadUsers() {
  const response = await fetch("https://jsonplaceholder.typicode.com/users");
  const data = await response.json();
  console.log(data);
}

fetch() делает запрос к серверу, а async/await помогает писать асинхронный код более понятным способом.

В реальном коде важно добавлять обработку ошибок через try...catch. 

async function loadData() {
  try {
    const response = await fetch("/api/data");
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

Частые ошибки новичков

Типичные ошибки в JavaScript
Ошибка Почему плохо Как лучше
Использовать var без необходимости Поведение менее предсказуемо Обычно использовать const и let
Путать == и === Можно получить неожиданные сравнения Предпочитать ===
Менять DOM слишком хаотично Код быстро становится запутанным Делать понятные шаги и разбивать код на функции
Игнорировать ошибки в запросах Трудно понять, почему код сломался Использовать try...catch
Писать всё в одном большом файле Код сложно читать и поддерживать Разделять логику на небольшие понятные части

Что важно запомнить

  • JavaScript отвечает за поведение и логику
  • const и let используются чаще всего
  • Функции помогают переиспользовать код
  • Массивы и объекты — основа работы с данными
  • DOM позволяет менять страницу через код
  • События помогают реагировать на действия пользователя
  • fetch и async/await нужны для работы с данными с сервера
