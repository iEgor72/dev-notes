Что такое Git

Git — это система контроля версий. Она помогает отслеживать изменения в файлах, сохранять историю проекта, возвращаться к прошлым версиям и работать над кодом безопаснее.

Git особенно полезен, когда ты развиваешь проект постепенно, пробуешь разные решения и хочешь не потерять рабочее состояние.

Git — это не сайт и не облако. Это инструмент для управления историей проекта на твоём компьютере.

Репозиторий

Репозиторий — это папка проекта, внутри которой Git хранит историю изменений. 

git init

Команда git init создаёт новый локальный Git-репозиторий в текущей папке.

Как начать проект 

mkdir my-project
cd my-project
git init

После этого можно создавать файлы, добавлять их в Git и делать первые коммиты.

Первая настройка Git 

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

Эти команды задают имя и email, которые будут указаны в твоих коммитах. 

git config --list

Показывает текущие настройки Git.

Без базовой настройки Git может не дать создать первый коммит.

status, add, commit 

git status
git add file.txt
git add .
git commit -m "Initial commit"
Основные команды для сохранения изменений
Команда Что делает
git status Показывает текущее состояние файлов
git add file.txt Добавляет в staging area конкретный файл
git add . Добавляет изменения из текущей папки
git commit -m "message" Создаёт сохранённую точку в истории

Коммит — это сохранённое состояние проекта с комментарием о том, что было сделано.

Для новичков часто безопаснее сначала использовать git add file.txt, а не git add ., чтобы случайно не добавить лишние файлы.

Просмотр изменений

git diff
git diff --staged

git diff показывает изменения, которые ещё не добавлены в staging area.

git diff --staged показывает изменения, которые уже были добавлены через git add.

Очень полезно перед коммитом: можно быстро проверить, что именно попадёт в историю.

История изменений

git log
git log --oneline

Эти команды показывают историю коммитов. Короткий вариант удобен для быстрого просмотра.

Ветки

Ветка позволяет работать над новой задачей отдельно от основной версии проекта. 

git branch
git branch feature-header
git switch feature-header
git switch -c feature-header
Команды для работы с ветками
Команда Что делает
git branch Показывает список веток
git branch name Создаёт новую ветку
git switch name Переключается на ветку
git switch -c name Создаёт ветку и сразу переключается на неё

Основная ветка проекта часто называется main.

Слияние веток

git switch main
git merge feature-header

Сначала переходят в основную ветку, потом вливают в неё изменения из другой ветки.

Если Git не может автоматически объединить изменения, возникает конфликт, который нужно решить вручную.

Удалённый репозиторий

Удалённый репозиторий — это версия проекта на сервисе вроде GitHub. Он нужен для резервного хранения кода и совместной работы. 

git remote add origin REPOSITORY_URL
git remote -v

origin — это стандартное имя для удалённого репозитория.

clone, pull, push

Команды для работы с удалённым репозиторием
Команда Что делает
git clone URL Копирует удалённый репозиторий на компьютер
git pull Загружает свежие изменения и объединяет их
git push Отправляет локальные коммиты в удалённый репозиторий 
git push -u origin main

Эта команда впервые связывает локальную ветку main с удалённой веткой.

fetch и pull

git fetch
git pull

git fetch загружает изменения из удалённого репозитория, но не сливает их в текущую ветку автоматически.

git pull загружает изменения и сразу пытается объединить их с текущей веткой.

fetch безопаснее для просмотра изменений, pull быстрее в повседневной работе.

Файл .gitignore

Файл .gitignore говорит Git, какие файлы и папки не нужно отслеживать. 

node_modules/
.env
dist/
.DS_Store

Обычно в .gitignore добавляют временные файлы, зависимости, сборку и приватные данные.

Временно убрать изменения 

git stash
git stash pop

git stash временно сохраняет незакоммиченные изменения и очищает рабочую директорию.

git stash pop возвращает эти изменения обратно.

Полезно, если нужно срочно переключиться на другую ветку, не делая лишний коммит.

Как отменять изменения 

git restore file.txt
git restore --staged file.txt
git commit --amend -m "New commit message"
Полезные команды для отмены и правок
Команда Что делает
git restore file.txt Отменяет локальные незакоммиченные изменения в файле
git restore --staged file.txt Убирает файл из staging area, но оставляет изменения в файле
git commit --amend Исправляет последний коммит

Обычный рабочий сценарий 

git status
git diff
git add .
git commit -m "Add new section"
git push

В повседневной работе чаще всего сценарий такой: проверить состояние репозитория, посмотреть изменения, добавить нужные файлы, сделать коммит и отправить результат в удалённый репозиторий.

Нужные команды на каждый день

Мини-шпаргалка повседневных команд
Команда Когда использовать
git status Проверить текущее состояние репозитория
git diff Посмотреть, что изменилось
git add file.txt Добавить конкретный файл в staging area
git commit -m "message" Сохранить изменения в историю
git pull Получить свежие изменения из удалённого репозитория
git push Отправить свои коммиты в удалённый репозиторий

Частые ошибки новичков

Типичные ошибки при работе с Git
Ошибка Почему плохо Как лучше
Редко делать коммиты Труднее понять историю и откатиться к нужной точке Делать небольшие понятные коммиты
Писать непонятные сообщения коммитов История становится бесполезной Писать ясно: что изменено
Отправлять в репозиторий лишние файлы Проект засоряется Использовать .gitignore
Работать всё время только в одной ветке Сложнее экспериментировать и вносить изолированные изменения Создавать отдельные ветки под задачи
Не смотреть git status Легко случайно закоммитить не то Почаще проверять состояние репозитория

Что важно запомнить

  • Git хранит историю изменений проекта
  • git status — одна из самых полезных команд
  • git diff помогает проверить изменения перед коммитом
  • git add подготавливает изменения, git commit сохраняет их в истории
  • Ветки помогают работать над задачами отдельно
  • push отправляет изменения, pull получает свежие
  • .gitignore помогает не засорять репозиторий лишними файлами
