Что такое React

React — это JavaScript-библиотека для создания пользовательских интерфейсов.

React помогает собирать страницу из отдельных частей: кнопок, карточек, форм, меню, модальных окон и других блоков.

React не заменяет HTML, CSS и JavaScript. Он помогает удобнее управлять интерфейсом и обновлять его при изменении данных.

Зачем нужен React

Когда страница становится интерактивной, обычный JavaScript-код начинает быстро разрастаться: нужно вручную искать элементы, менять текст, классы, скрывать блоки и следить за состоянием.

React упрощает эту задачу и даёт несколько важных преимуществ:

  • интерфейс можно разбить на переиспользуемые компоненты
  • данные и отображение связаны между собой
  • страница обновляется автоматически при изменении состояния
  • сложные интерфейсы становятся проще поддерживать

React особенно полезен там, где много переключателей, фильтров, форм, вкладок, карточек и динамических блоков.

Как React работает

В React интерфейс описывается как функция от данных. Ты говоришь, что должно быть на экране, а React сам обновляет DOM, когда данные меняются.

Упрощённая логика работы React
Шаг Что происходит
1 Компонент получает данные
2 React строит интерфейс на основе этих данных
3 Пользователь нажимает кнопку или вводит текст
4 Состояние меняется
5 React обновляет нужную часть интерфейса

Что такое JSX

JSX — это синтаксис, который внешне похож на HTML, но используется внутри JavaScript.

С помощью JSX удобно описывать, как должен выглядеть интерфейс. 

function App() {
  return (
    <section>
      <h1>Привет, React</h1>
      <p>Это мой первый компонент.</p>
    </section>
  );
}

JSX похож на HTML, но это не чистый HTML. Например, вместо class используется className.

Компоненты

Компонент — это самостоятельная часть интерфейса. Например, карточка товара, кнопка, шапка сайта или форма поиска.

В React интерфейс строится из компонентов, которые можно вкладывать друг в друга. 

function Header() {
  return <header>Мой сайт</header>;
}

function App() {
  return (
    <div>
      <Header />
      <main>Контент страницы</main>
    </div>
  );
}

Названия компонентов пишут с большой буквы. Это важное правило в React.

Props

Props — это входные данные компонента. Через них родительский компонент передаёт значения дочернему. 

function Welcome(props) {
  return <h2>Привет, {props.name}!</h2>;
}

function App() {
  return <Welcome name="Егор" />;
}
Что важно знать о props
Свойство Что означает
Передаются сверху вниз От родительского компонента к дочернему
Только для чтения Нельзя изменять props внутри компонента
Подходят для настройки Через props удобно менять текст, данные и поведение

State

State — это внутренние данные компонента, которые могут меняться во время работы.

Например: открыто ли меню, какой текст ввёл пользователь, активна ли вкладка, сколько раз нажали на кнопку. 

const [count, setCount] = useState(0);

Здесь count — текущее значение, а setCount — функция для его обновления.

Когда state меняется, React заново отрисовывает компонент с новым значением.

События

В React можно обрабатывать клики, ввод текста, отправку формы и другие действия пользователя. 

function ButtonExample() {
  function handleClick() {
    alert("Кнопка нажата");
  }

  return <button onClick={handleClick}>Нажми</button>;
}

Имена обработчиков часто начинаются с handle, а сами события записываются в camelCase: onClick, onChange, onSubmit.

Условный рендеринг

В React можно показывать разный интерфейс в зависимости от условий. 

function Status({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <p>Вы вошли</p> : <p>Войдите в аккаунт</p>}
    </div>
  );
}

Это удобно для показа кнопок, сообщений, ошибок, загрузки и разных состояний интерфейса.

Списки и key

Когда нужно вывести массив данных, обычно используют map(). 

function TechList() {
  const items = ["HTML", "CSS", "React"];

  return (
    <ul>
      {items.map((item) => (
        <li key={item}>{item}</li>
      ))}
    </ul>
  );
}

Атрибут key нужен React, чтобы понимать, какой именно элемент списка изменился.

Не используй случайные значения для key. Лучше давать стабильный уникальный идентификатор.

useState

useState — это хук, который позволяет хранить и изменять состояние внутри функционального компонента. 

import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Счётчик: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Увеличить
      </button>
    </div>
  );
}

Что хранится

count содержит текущее значение состояния.

Что меняет

setCount обновляет состояние и запускает перерисовку компонента.

React и обычный JavaScript

Обе технологии нужны. React не отменяет JavaScript, а строится на нём.

Обычный JavaScript

const box = document.getElementById("box");
box.textContent = "Новый текст";

React

function Message() {
  const [text, setText] = useState("Привет");

  return <p>{text}</p>;
}

В обычном JavaScript ты чаще вручную меняешь DOM. В React ты меняешь данные, а интерфейс обновляется сам.

Частые ошибки новичков

Ошибки при изучении React
Ошибка Почему это плохо Как лучше
Писать class вместо className JSX использует свои правила записи атрибутов Использовать className
Менять state напрямую React не увидит корректное обновление Использовать функцию обновления состояния
Забывать key в списках React сложнее отслеживать элементы Указывать стабильный уникальный key
Называть компонент с маленькой буквы React воспримет его как обычный HTML-тег Писать компоненты с большой буквы

Плохо

function button() {
  return <button>Нажми</button>;
}

Лучше

function Button() {
  return <button>Нажми</button>;
}

Плохо

count = count + 1;

Лучше

setCount(count + 1);

Что важно запомнить

  • React нужен для создания интерфейсов
  • Интерфейс в React строится из компонентов
  • JSX похож на HTML, но это часть JavaScript-кода
  • Props передают данные в компонент
  • State хранит изменяемые данные компонента
  • При изменении state React обновляет интерфейс
  • Спискам нужен key
  • useState — один из базовых хуков React

Мини-пример React-компонента 

import { useState } from "react";

function FlexModeDemo() {
  const [mode, setMode] = useState("flex");

  return (
    <section>
      <h2>Display demo</h2>

      <div>
        <button onClick={() => setMode("block")}>block</button>
        <button onClick={() => setMode("flex")}>flex</button>
        <button onClick={() => setMode("grid")}>grid</button>
      </div>

      <div className={`demo demo--${mode}`}>
        <div>1</div>
        <div>2</div>
        <div>3</div>
      </div>
    </section>
  );
}

Это уже похоже на тот тип интерактивных демо, который ты хочешь добавлять на сайт: кнопка меняет состояние, а интерфейс сразу перестраивается.
