Что такое React

React — JavaScript-библиотека для создания пользовательских интерфейсов. Она используется для построения интерактивных веб-приложений, где элементы страницы должны изменяться без полной перезагрузки.

Основная идея React заключается в построении интерфейса из небольших независимых компонентов. Каждый компонент отвечает за отдельную часть интерфейса и может переиспользоваться в разных местах приложения.

React не заменяет HTML, CSS и JavaScript. Он работает поверх JavaScript и помогает удобнее управлять состоянием интерфейса и обновлением DOM.

Зачем используется React

В сложных интерфейсах обычный JavaScript-код может быстро становиться трудным для поддержки. React решает эту проблему за счёт компонентной архитектуры и автоматического обновления интерфейса.

  • позволяет разбивать интерфейс на независимые компоненты
  • обновляет только изменённые части страницы
  • упрощает поддержку и масштабирование интерфейса
  • позволяет переиспользовать UI-компоненты

React особенно полезен в интерфейсах с большим количеством интерактивных элементов: форм, фильтров, переключателей, списков и динамических данных.

Как React работает

React описывает интерфейс как функцию от данных. Компонент получает данные и на их основе формирует разметку.

Когда данные изменяются, React автоматически обновляет соответствующую часть интерфейса.

Упрощённая схема работы React
Шаг Описание
1 Компонент получает данные
2 На основе данных строится интерфейс
3 Пользователь выполняет действие
4 Состояние изменяется
5 React обновляет нужную часть DOM

JSX

JSX — синтаксис, используемый в React для описания интерфейса. Он напоминает HTML, но фактически является расширением JavaScript.

JSX позволяет описывать структуру интерфейса прямо внутри JavaScript-кода. 

function App() {
  return (
    <section>
      <h1>Привет, React</h1>
      <p>Это простой JSX-пример.</p>
    </section>
  );
}

JSX отличается от HTML. Например, вместо атрибута class используется className.

Компоненты

Компонент — это самостоятельная часть интерфейса. Например, кнопка, карточка, форма, шапка сайта или панель навигации.

Приложение на React обычно состоит из дерева компонентов, где одни компоненты могут включать в себя другие. 

function Header() {
  return <header>Developer Notes</header>;
}

function App() {
  return (
    <div>
      <Header />
      <main>Содержимое страницы</main>
    </div>
  );
}

Названия React-компонентов пишутся с большой буквы.

Props

Props — это входные данные компонента. Через них родительский компонент передаёт значения дочернему. 

function Welcome(props) {
  return <h2>Привет, {props.name}!</h2>;
}

function App() {
  return <Welcome name="Egor" />;
}
Основные свойства props
Свойство Описание
Передаются сверху вниз От родительского компонента к дочернему
Только для чтения Компонент не должен изменять собственные props
Используются для настройки Через props удобно передавать текст, числа и данные

State

State — это внутренние данные компонента, которые могут изменяться во время работы.

State используется для хранения текущего состояния интерфейса: открыто ли меню, активна ли вкладка, сколько раз нажали кнопку, какой текст введён в поле. 

const [count, setCount] = useState(0);

Здесь count — текущее значение, а setCount — функция, которая обновляет состояние.

После изменения state React заново отрисовывает компонент с актуальными данными.

События

В React можно обрабатывать клики, ввод текста, отправку формы и другие действия пользователя. 

function ButtonExample() {
  function handleClick() {
    alert("Кнопка нажата");
  }

  return <button onClick={handleClick}>Нажми</button>;
}

Названия событий в React записываются в формате camelCase: onClick, onChange, onSubmit.

Условный рендеринг

React позволяет отображать разные части интерфейса в зависимости от условий. 

function Status({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <p>Вход выполнен</p> : <p>Требуется авторизация</p>}
    </div>
  );
}

Такой подход используется для сообщений, загрузки, ошибок, кнопок и различных состояний приложения.

Списки и key

Для вывода массива данных в React обычно используется метод map(). 

function TechList() {
  const items = ["HTML", "CSS", "React"];

  return (
    <ul>
      {items.map((item) => (
        <li key={item}>{item}</li>
      ))}
    </ul>
  );
}

Атрибут key помогает React корректно отслеживать элементы списка при обновлении.

Для key лучше использовать стабильное уникальное значение, а не случайные данные.

useState

useState — один из базовых хуков React. Он позволяет добавлять состояние в функциональные компоненты. 

import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Счётчик: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Увеличить
      </button>
    </div>
  );
}

Текущее значение

count хранит текущее состояние компонента.

Функция обновления

setCount изменяет состояние и запускает перерисовку.

React и обычный JavaScript

React основан на JavaScript и не заменяет его. Разница заключается в подходе к управлению интерфейсом.

Обычный JavaScript

const box = document.getElementById("box");
box.textContent = "Новый текст";

React

function Message() {
  const [text, setText] = useState("Привет");

  return <p>{text}</p>;
}

В обычном JavaScript чаще изменяют DOM напрямую. В React обычно изменяют данные, а интерфейс обновляется автоматически.

Частые ошибки новичков

Типичные ошибки при изучении React
Ошибка Почему плохо Как правильно
Использовать class вместо className JSX использует свои правила записи атрибутов Использовать className
Изменять state напрямую React не обработает обновление корректно Использовать функцию обновления состояния
Не указывать key в списках React сложнее отслеживать элементы Использовать стабильный уникальный key
Называть компонент с маленькой буквы React воспримет его как HTML-тег Имена компонентов писать с большой буквы

Неправильно

function button() {
  return <button>Нажми</button>;
}

Правильно

function Button() {
  return <button>Нажми</button>;
}

Неправильно

count = count + 1;

Правильно

setCount(count + 1);

Что важно запомнить

  • React используется для создания пользовательских интерфейсов
  • Интерфейс строится из компонентов
  • JSX похож на HTML, но является частью JavaScript
  • Props передают данные в компонент
  • State хранит изменяемое состояние компонента
  • При изменении state React обновляет интерфейс
  • Для списков нужен атрибут key
  • useState — базовый хук для работы с состоянием

Базовый пример React-компонента 

import { useState } from "react";

function FlexModeDemo() {
  const [mode, setMode] = useState("flex");

  return (
    <section>
      <h2>Display demo</h2>

      <div>
        <button onClick={() => setMode("block")}>block</button>
        <button onClick={() => setMode("flex")}>flex</button>
        <button onClick={() => setMode("grid")}>grid</button>
      </div>

      <div className={`demo demo--${mode}`}>
        <div>1</div>
        <div>2</div>
        <div>3</div>
      </div>
    </section>
  );
}

Это простой пример React-компонента. Состояние mode хранит выбранный режим отображения, а кнопки изменяют это значение через setMode. Когда состояние обновляется, React автоматически перерисовывает интерфейс.
